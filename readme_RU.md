## 📋 Описание

**FW_NULL** - это bash-скрипт для экстренного сброса настроек файрвола (iptables/UFW) до стандартных значений ACCEPT ALL. Создан специально для ситуаций, когда администратор случайно блокирует себе доступ к серверу (например, закрывает 22 порт) и теряет возможность подключиться по SSH.

Скрипт полностью очищает все правила iptables, отключает UFW и другие сервисы управления файрволом, восстанавливая доступ к серверу.

## 🎯 Для чего это нужно?

- **Экстренное восстановление доступа** - если случайно заблокировали SSH (порт 22)
- **Сброс "сломанных" правил** - когда файрвол настроен некорректно
- **Подготовка сервера к перенастройке** - нужно начать с чистого листа
- **Миграция серверов** - перенос конфигурации на новое оборудование

## ✨ Возможности

- ✅ **Автоматическое создание бэкапов** - все текущие правила сохраняются перед сбросом
- ✅ **Полное отключение UFW** - сервис останавливается и отключается из автозагрузки
- ✅ **Остановка persistence-сервисов** - netfilter-persistent, iptables-persistent, firewalld
- ✅ **Сброс всех IPv4 правил** - очистка всех таблиц (filter, nat, mangle, raw, security)
- ✅ **Сброс всех IPv6 правил** - если IPv6 включен
- ✅ **Установка политик ACCEPT** - INPUT, FORWARD, OUTPUT = ACCEPT
- ✅ **Проверка SSH доступности** - убеждается, что SSH работает
- ✅ **Отображение открытых портов** - показывает, какие сервисы доступны
- ✅ **Цветной вывод** - удобная навигация по шагам выполнения
- ✅ **Подробное логирование** - все действия записываются в `/var/log/fw_null.log`
- ✅ **Опциональное сохранение** - можно сохранить пустые правила после перезагрузки

## 📖 Использование

### Базовый запуск

```bash
sudo ./fw_null
```

Скрипт выполнит следующие шаги:
1. Проверит права root
2. Создаст директорию для бэкапов
3. Сохранит текущие правила
4. Отключит UFW
5. Остановит persistence-сервисы
6. Сбросит все IPv4 правила
7. Сбросит все IPv6 правила
8. Покажет открытые порты
9. Проверит SSH доступность
10. Спросит о сохранении пустых правил

## 🔧 Требования к системе

- **Команды**: iptables, ip6tables (опционально)

## ⚠️ Важные замечания

1. **Скрипт должен запускаться с sudo** - без прав root он не сработает
2. **Бэкапы создаются автоматически** - можно восстановиться, если что-то пошло не так
3. **После сброса все порты открыты** - сервер становится уязвимым, настройте файрвол заново
4. **SSH должен быть настроен** - скрипт проверяет, работает ли SSH после сброса
5. **Настройки не сохраняются автоматически** - спросит, хотите ли вы сохранить пустые правила

## 🔄 Восстановление из бэкапа

Если нужно вернуть старые правила:

```bash
# Посмотреть доступные бэкапы
ls -la /root/fw_backups/

# Восстановить IPv4 правила
sudo iptables-restore < /root/fw_backups/20240213_143022/iptables.v4

# Восстановить IPv6 правила
sudo ip6tables-restore < /root/fw_backups/20240213_143022/ip6tables.v6

# Включить UFW обратно (если использовался)
sudo ufw enable
```

## 🐛 Решение проблем

### "Порт 22 все еще не доступен"
```bash
# Проверить статус SSH
systemctl status sshd

# Проверить, слушает ли SSH порт 22
ss -tlnp | grep 22

# Перезапустить SSH
systemctl restart sshd
```

### "Не могу сохранить правила"
```bash
# Установить netfilter-persistent
apt-get install iptables-persistent netfilter-persistent
# или
yum install iptables-services

# Запустить сохранение вручную
iptables-save > /etc/iptables/rules.v4
```
